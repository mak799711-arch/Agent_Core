# Инструкция по деплою AgentCore на NAS (Synology / Docker)

Так как мы используем домашний/офисный NAS отца в качестве сервера, самый надежный способ запустить там AgentCore — через **Docker**. Мы уже подготовили `Dockerfile` (сборка приложения) и `docker-compose.yml` (настройки запуска).

## Шаг 1: Подготовка на NAS
1. Убедись, что на NAS установлен пакет **Container Manager** (или **Docker** на старых версиях DSM).
2. Зайди в **File Station** и создай папку для проекта, например: `/docker/agentcore`.
3. Скопируй в эту папку файлы проекта (или хотя бы `docker-compose.yml`, а сам образ мы можем собрать локально и загрузить, либо собирать прямо там, если NAS мощный). 
*Совет: Лучше всего перенести туда весь исходный код через Git или FTP.*

## Шаг 2: Настройка переменных окружения
В папке `/docker/agentcore` создай файл `.env` и впиши туда ключи из твоего Supabase:
```env
NEXT_PUBLIC_SUPABASE_URL=твой_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=твой_anon_key
SUPABASE_SERVICE_ROLE_KEY=твой_service_key
```

## Шаг 3: Запуск
Если у тебя есть SSH доступ к NAS:
```bash
cd /volume1/docker/agentcore
docker-compose up -d --build
```
Если ты делаешь через интерфейс DSM (Container Manager):
1. Открой **Container Manager** -> **Проекты** -> **Создать**.
2. Укажи путь к папке `/docker/agentcore`.
3. Выбери "Использовать существующий docker-compose.yml".
4. Запусти проект.

## Шаг 4: Доступ извне
Приложение запустится на порту `3000`. Если локальный IP твоего NAS `192.168.1.100`, то приложение будет доступно по `http://192.168.1.100:3000`.
Для вывода его в интернет тебе понадобится настроить **Обратный прокси (Reverse Proxy)** в панели управления Synology:
- Источник: твой домен (например, `app.agentcore.com`) с HTTPS (порт 443).
- Назначение: `localhost`, порт `3000`.
